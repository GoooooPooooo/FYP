# 🤖 Система удалённого управления промышленным роботом MELFA RV-2AJ

## 📖 Как это работает
Система реализует двунаправленный обмен данными между клиентом и роботом через облачный MQTT-брокер:
1. **Клиент** (IoT MQTT Panel) публикует команду в топик `.../sub`.
2. **ESP32** получает сообщение по Wi-Fi/MQTT и транслирует его через UART (RS-232) контроллеру робота.
3. **Робот** выполняет команду и возвращает текстовый ответ.
4. **ESP32** считывает ответ по UART и публикует его в топик `.../pub`.
5. **Клиент** отображает статус/ответ в реальном времени.

---

## 🔌 Схема подключения кабеля

### 📡 Последовательность соединений
```
[Контроллер CR1]          [Конвертер RS-232↔TTL]           [ESP32]
   DB-25 порт                Сторона RS-232                 Пины GPIO
       │                          │                            │
 TXD (пин 2) ──────────────────→ RXD ──────────────────────→ GPIO16 (RX)
 RXD (пин 3) ──────────────────→ TXD ──────────────────────→ GPIO17 (TX)
 GND (пин 7) ──────────────────→ GND ──────────────────────→ GND
       │                          │                            │
       └──────────────────────────┴────────────────────────────┘
```

### 📋 Таблица распиновки
| Сторона RS-232 (Робот/Конвертер) | Сторона TTL (ESP32) | Назначение |
|:---|:---|:---|
| `RXD` (вход конвертера) | `GPIO17` | Передача данных (ESP32 TX → Робот RX) |
| `TXD` (выход конвертера) | `GPIO16` | Приём данных (Робот TX → ESP32 RX) |
| `GND` | `GND` | Общая земля **(обязательно!)** |
| `VCC 5V` (опционально) | `VIN` | Питание конвертера (при отсутствии внешнего БП) |

> ⚠️ **Важно:** Линии `TX` и `RX` подключаются **перекрёстно**. Без соединения `GND` обмен данными невозможен из-за разности потенциалов.

## 🛠 Требования
- **Аппаратная часть:** ESP32, конвертер RS-232 ↔ TTL (MAX232/SP3232), кабель DB25↔DB9 (при необходимости), промышленный робот MELFA RV-2AJ (контроллер CR1).
- **Программная часть:** Прошивка MicroPython для ESP32, Thonny IDE, IoT MQTT Panel, доступ к Wi-Fi 2.4 ГГц.
- **Сетевые параметры:** Доступ к `srv2.clusterfly.ru:9991` (TCP).

## 🚀 Установка и запуск

### 1. Подготовка ESP32
1. Прошейте плату актуальной прошивкой MicroPython через `esptool.py`.
2. Подключите ESP32 к ПК, откройте **Thonny IDE** и выберите интерпретатор `MicroPython (ESP32)`.

### 2. Настройка кода (`main.py`)
- В начале файла укажите реальные данные Wi-Fi:
  ```python
  WIFI_SSID = "ВАША_СЕТЬ"
  WIFI_PASSWORD = "ВАШ_ПАРОЛЬ"
  ```
- Параметры MQTT уже настроены под учебный стенд. При смене аккаунта обновите `MQTT_USER`, `MQTT_TOPIC_PUB` и `MQTT_TOPIC_SUB`.
- Сохраните файл на плате как `main.py` (автозапуск при подаче питания).

### 3. Аппаратная сборка
- Подключите конвертер к ESP32 согласно схеме выше.
- Подключите сторону RS-232 к COM-порту контроллера робота.
- Убедитесь, что все соединения надёжны, а контакты `TX/RX` не перепутаны.

### 4. Запуск
- В Thonny нажмите **Run** или перезагрузите плату.
- В консоли должны появиться сообщения:
  ```
  Wi-Fi подключен: ('192.168.x.x', '255.255.255.0', ...)
  Подключено к MQTT брокеру
  Подписались на топик: user_816e63ee/sub
  ```

## 📜 Справочник команд MOVEMASTER
| Команда | Описание |
|:---|:---|
| `GO` | Открыть захват |
| `GC` | Закрыть захват |
| `SP <0-30>` | Установка скорости (0–30) |
| `MP X,Y,Z,A,B,C` | Линейное перемещение в точку |
| `DJ <ось>,<угол>` | Поворот отдельной оси (Joint) |
| `NT` | Возврат в нулевое положение |
| `RN` / `ED` | Запуск программы / Конец программы |

## ⚠️ Важные примечания
- **Параметры UART:** `9600 бод, 8 бит, чётность: None, стоп-биты: 2` (`8N2`). В коде задано `uart.init(baudrate=9600, bits=8, parity=0, stop=2)`.
- **Задержки:** В коде используются `time.sleep(1)` для стабильного чтения буфера UART. При отладке в идеальных условиях их можно уменьшить до `0.2–0.5`.
- **Безопасность:** При первом запуске держите руку на кнопке `EMERGENCY STOP`. Убедитесь, что в рабочей зоне робота нет людей и препятствий.
- **Отладка:** При отсутствии ответов проверяйте: статус Wi-Fi, корректность перекрёстного подключения TX/RX, общую землю и положение ключей контроллера (`AUTO(ext)` + `DISABLE`).
